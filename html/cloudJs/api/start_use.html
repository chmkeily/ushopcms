开始使用

cloudjs包含20个组件，只需要引入jquery和cloudjs.js就可以调用cloudjs的所有组件；
组件调用方式为：cloudjs(selector).组件名([options]) 或 cloudjs.组件名([options])

目录结构

  • api ———— 包含该网站除首页以外的所有页面，example目录则包含所有的演示实例
  • js ———— 包含cloudjs框架中所有组件的js及jquery,cloudjs.js,复制到粘贴板组件的swf文件
  • css ———— css中包含6种肤色的目录，里面分别存放各肤色的css，cloudjs默认引入blue目录下的css
  • images ———— 包含组件中用到的所有图片
  • gulp ———— 包含less文件，阅读help.txt，可轻松生成任意指定的肤色并压缩js和css

如何换肤

换肤分两种情况
第1种情况，我不想用blue肤色，我想用red肤色，可以把red目录下的css复制到blue目录下，替换原有的，可以修改cloudjs.js第7行为defaultThemes='red', 也可以引入cloudjs时指定data-themes="red"。
第2种情况，我的网站使用了多种肤色，需要点击按钮的时候动态改变，调用cloudjs.themes('肤色目录名');动态换肤，引入cloudjs时，加上data-themes指定肤色，如：
<script src="http://cloudjs.wsd.com/cloudJs/js/cloudjs.js" data-themes="red"></script>。

变更组件时如何让老用户看到最新的效果

当你修改cloudjs中的css或者js时，由于浏览器缓存，发布到服务器后，老用户还是会看到更新前的状态，尤其是修改bug的情况下，及时更新显得尤为重要。
方法：修改js目录下version.js中的key值，这里的key值就是组件js和css加载时自带的后缀，后缀变了，文件自然也就及时刷新了。
