validate格式校验组件

用于页面输入框的格式校验

cloudjs(selector).validate([options]) options={'属性名':'属性值',...}

属性

属性名 类型 默认值 说明
checkType string empty 元素默认的校验类型，元素可以通过data-check来设置元素的检验类型
warn boolean true 是否显示错误信息
errmsgType number 1 页面错误信息提示的方式，1是弹tips，2是直接跟在元素后面或在指定的区域显示，如果设置了2，元素可以通过data-wraper属性设置错误提示显示的地方，如将错误信息显示在一个id为myerr的元素上，则data-wraper="#myerr"（warn为true时生效）
posiiton string up Tips在元素的提示位置，up表示上方，left表示左方，right表示右方，down表示下方，优先是down，组件会自动计算出现的位置，以确保出现的提示不会超出页面边界（元素也可以通过设置data-position来设置显示的位置）（warn为true并且errmsg为1时生效）
delay number 0 错误提示多久自动消失，0表示不自动消失，单位ms（wran为true时生效）
rule null 组件自己的默认校验规则(非空empty、日期date、时间time、链接url、数字num、整数inum、正整数uinum、非负整数ginum、单ip、多ips、邮箱email、手机号码phone、命名naming、最大长度maxlen[x]、最小长度minlen[x]、) 自定义校验规则和错误信息，每种校验类型对应rule的一个属性，该属性值对应一个设置对象，这个对象有三个属性，errmsg,format和check，errmsg对应一个方法,该方法返回一个错误提示字符串，format对应多种格式,用于比如校验多种日期格式的选择,check对应一个校验方法,该方法返回是否校验通过，errmsg和check两个方内部都通过传入的参数获取到当前被校验的输入框对象
success function() 校验成功后调用的方法
fail function(errObj) 校验失败时调用的方法，会传入校验失败的详细信息，把校验失败的元素和错误提示信息封装成数组的一个个对象，然后将数组传给fail方法

cloudjs(selector).validate('方法名')

方法

方法名 参数 说明
setData 重新设置属性值，分永久设置和临时设置，永久设置会作用于后面重新弹出的提示信息，而临时设置作用于当前弹出的提示信息
