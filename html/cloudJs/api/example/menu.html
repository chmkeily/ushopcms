menu菜单组件

cloudjs(selector).menu([options])

属性

属性名 类型 默认值 说明
第一种类型菜单的参数(示例1和示例2)
data jsonArray 必填，下拉菜单的数据，数据格式为data/menu_data.js，详见示例1
type string 下拉菜单的类型，横向或纵向，默认为横向(见示例1)，type等于vertical为纵向菜单，详见示例2
isPreLoad boolean false 是否提前加载菜单数据，false表示鼠标移到菜单上才加载数据，true表示调用组件时就加载数据，如果对菜单内容加click事件，则需要设置isPreLoad为true，或者采用live('click')，详见示例2
css object { top1:,left1,top2:,left2: } top1,left1为第一级菜单相对于父节点li的top和left值，top2,left2为第一级以后的菜单相对于父节点li的top和left值，当菜单与菜单直接的距离没有衔接上时(即上一级ul设置padding或border时)，可以通过这4个值来调节距离，详见示例3
第二种类型菜单的参数(见示例4和示例5)
relativeTop string 隐藏内容相对top，默认效果见示例4，relativeTop等于current的效果见示例5
css objcet { width:500, height:, top:, left: } width,height,top,left分别为隐藏元素的宽度,高度,top和left值,一般情况下只需要指定width，其他三个值组件会默认计算出来
第三种类型菜单的参数(见示例6和示例7)
position string 隐藏元素相对展示元素的位置,默认为左对齐,值为right,center分别表示右对齐和居中对齐，见示例7
第四种类型菜单的参数(见示例8，示例9，示例10)
type string 必填，必须指定type的值为tree，才会调用第四种类型菜单
data jsonArray 必填，如果页面用ul写了第一级菜单，则data为二维数组，见示例8，如果页面没有写第一级菜单，则data为一维数组，详见示例9
isHideOther boolean true 点击节点时，是否隐藏同级节点的子节点，见示例10
selectedName string href 默认指定选中节点时要比较对象的name示例10
selectedValue string 默认指定选择的节点时比较的值，如果设定selectedValue:'paging.html',那么初始化组件时，在data中只有寻找到obj[selectedName]=='paging.html'的对象，就会把该对象生成的a元素加上class="selected"的标签，详见示例10
onClick function(obj) 点击菜单节点时触发的事件，返回当前点击的节点，详见示例8
